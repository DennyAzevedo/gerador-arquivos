# Documentação — Gerador de Artigos para WordPress

Documentação viva do projeto. **Todas as 10 fases do roteiro foram concluídas** — o MVP está operacional, com evoluções posteriores (Gemini, WordPress mock).

> Projeto desenvolvido integralmente com o [Cursor](https://cursor.com), sem intervenção manual no código.

## Índice

| Documento | Descrição |
| --------- | --------- |
| [01 — Stack Tecnológica](./01-stack.md) | Stack sugerida e justificativa de cada escolha (backend, frontend, banco, infra). |
| [02 — Planejamento e Arquitetura](./02-planejamento.md) | Arquitetura geral, estrutura de pastas, modelo de dados e endpoints. |
| [03 — Regras de Codificação](./03-regras-codificacao.md) | Resumo das regras (`.cursor/rules`) que orientam toda a geração de código. |
| [04 — Etapas de Execução](./04-etapas-execucao.md) | Roteiro de fases com checklists, atualizado a cada passo concluído. |
| [05 — Prompts Utilizados](./05-prompts.md) | Registro cronológico dos prompts enviados ao Cursor na construção do projeto. |

## Como usar esta documentação

1. Antes de iniciar uma fase, consulte o documento [04 — Etapas de Execução](./04-etapas-execucao.md).
2. Toda geração de código deve respeitar o documento [03 — Regras de Codificação](./03-regras-codificacao.md).
3. Ao concluir uma etapa, atualize o status e o histórico no documento 04.
4. Para entender o fluxo de desenvolvimento orientado por IA, consulte [05 — Prompts Utilizados](./05-prompts.md).
